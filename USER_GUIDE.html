Overview

This guide helps users get started with GABM, find documentation, and get support. Documentation was updated in release 0.1.1 for clarity and completeness. It will be updated as more features and configuration options are added.

Getting Started

  • See SETUP_GUIDE_USER.md for installation and setup instructions.

  • Explore the documentation in the docs/ directory or online via GitHub Pages.

Key Documentation Files for Users

The following Markdown files are most relevant for users:

  • README.md: Project overview and quick start instructions.

  • USER_GUIDE.md: Detailed user instructions for installing, configuring, and running GABM.

  • SETUP_GUIDE_USER.md: Step-by-step environment setup for users.

  • API_KEYS.md: Information on required API keys for LLMs and how to configure them.

Refer to these files for guidance on using GABM. Additional documentation is available for developers and contributors.

Support & Feedback

  • If you encounter issues or have questions, please submit a GitHub Issue in the repository.

  • For general feedback or feature requests, use GitHub Issues or contact maintainers via GitHub.

Configuration & Running Models

  • (To be updated as features are added)

  • In future releases, users will be able to configure and run models directly from the interface or command line.

Troubleshooting

This section is for documenting common issues users may encounter when installing, configuring, or running GABM, along with solutions and workarounds. If you experience a problem, check here for guidance or updates. As the project evolves, troubleshooting tips and frequently asked questions will be added to help users resolve issues quickly.

Managing Logs and Caches

GABM creates logs and caches (such as prompt/response caches for LLM services) that can grow large over time. These files are not stored in the GitHub repository. Tools for tidying up logs and caches are planned for a future release. More details will be provided as these features are implemented.

Additional Resources

  • See README.md for project overview

  • If you encounter difficulties, please peruse reported issues and comment on an open issue or open an issue.
